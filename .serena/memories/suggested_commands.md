# 推奨コマンド一覧

このドキュメントは、Perfect Shuffle プロジェクトで開発する際に知っておくべき重要なコマンドをまとめたものです。

## 開発コマンド

### 開発サーバーの起動
```bash
npm run dev
```
- Vite開発サーバーを起動
- ホットモジュールリプレースメント（HMR）が有効
- 通常は`http://localhost:5173`でアクセス可能

### 本番ビルド
```bash
npm run build
```
- TypeScriptのコンパイル（`tsc -b`）と本番用バンドルの生成
- `dist/`ディレクトリに出力
- デプロイ前に必ず実行して動作確認

### 本番ビルドのプレビュー
```bash
npm run preview
```
- ビルドした本番用ファイルをローカルで確認
- 本番環境と同等の動作をテスト可能

## コード品質チェック

### Lintチェック
```bash
npm run lint
```
- Biomeでコード品質をチェック
- エラーや警告を表示
- **コミット前に必ず実行すること**

### コードフォーマット
```bash
npm run format
```
- Biomeでコードを自動フォーマット
- ダブルクオート、インデント2スペース、行幅100に整形
- **コミット前に必ず実行すること**

## テスト

### テスト実行（Watch モード）
```bash
npm run test
```
または
```bash
npm test
```
- Vitestをwatchモードで起動
- ファイル変更を検知して自動的に再実行
- 開発中は常に実行しておくことを推奨

### テスト実行（一回のみ）
```bash
npm run test -- --run
```
- watchモードなしで一度だけテストを実行
- CI/CDやコミット前の確認に使用

### テストカバレッジ
```bash
npm run test -- --coverage
```
- カバレッジレポートを生成
- どの部分がテストされているかを確認可能

## Git操作

### Git標準コマンド
```bash
git status              # 変更状況を確認
git add .              # すべての変更をステージング
git commit -m "message" # コミット
git push               # リモートにプッシュ
git pull               # リモートから最新を取得
git checkout -b branch # 新しいブランチを作成して切り替え
```

### GitHub CLI（gh）
```bash
gh issue view <番号>    # イシューの詳細を表示
gh pr create           # プルリクエストを作成
gh repo view           # リポジトリ情報を表示
```

## Linux/WSL2 標準コマンド

### ファイル操作
```bash
ls              # ファイル一覧
cd <ディレクトリ> # ディレクトリ移動
pwd             # 現在のディレクトリパス
cat <ファイル>   # ファイル内容を表示
head <ファイル>  # ファイルの先頭を表示
tail <ファイル>  # ファイルの末尾を表示
```

### 検索
```bash
grep <パターン> <ファイル>        # ファイル内検索
find <ディレクトリ> -name <パターン> # ファイル名検索
```

## タスク完了時の推奨フロー

タスクを完了したら、以下の順序でコマンドを実行してください：

1. **フォーマット**: `npm run format`
2. **テスト**: `npm run test -- --run`
3. **Lintチェック**: `npm run lint`
4. **ビルド確認**: `npm run build`
5. **コミット**: すべてのチェックが通過したら、Git操作を実行

この順序を守ることで、コードの品質を保ち、バグを早期に発見できます。
